# Spring Boot Learning and Teaching Kit

A beginner friendly and classroom ready Spring Boot handbook built for learning, teaching, and growing toward junior backend readiness.

This repository is designed for:

- Java learners moving into backend development
- instructors teaching Spring Boot for the first time
- developers using NetBeans 2024 and Java 23
- learners who want simple explanations before advanced ideas

## Main goal

This repo helps you do two things at the same time:

1. learn Spring Boot properly
2. teach Spring Boot clearly

It is not just a note dump.
It is structured like a guided handbook with lessons, assessments, assignments, answer guides, and runnable example projects.

## Module overview

### Module 01: Foundations
Covers backend basics, Spring Boot meaning, HTTP and REST basics, Maven, project structure, and first endpoints.

### Module 02: Spring Web Core
Covers request parameters, path variables, request bodies, POST, `ResponseEntity`, status codes, controller differences, and properties basics.

### Module 03: Dependency Injection and App Structure
Covers IoC, DI, beans, ApplicationContext, services, constructor injection, layered architecture, and refactoring logic out of controllers.

### Module 04: Data and CRUD
Covers JPA, entities, repositories, CRUD, validation, exceptions, DTOs, and a complete beginner CRUD project.

### Module 05: Configuration, Testing, and Docs
Covers deeper configuration, profiles, logging, testing basics, and Swagger or OpenAPI ideas.

### Module 06: Industry Readiness
Covers Spring Security basics, authentication, authorization, packaging, deployment, Docker, capstone thinking, and interview recap.

## How to use this repo

### If you are learning alone
1. Read `START_HERE.md`
2. Read `docs/INDEX.md`
3. Follow the roadmap
4. Study one module at a time
5. Run the matching example project
6. Answer the assessments and assignments yourself

### If you are teaching
1. Read `docs/teacher-edition/TEACHER-GUIDE.md`
2. Use each lesson as a classroom note
3. Use the assessments for class checks
4. Use the assignments for homework or labs
5. Use the answer guides for review and marking

### If you want project practice
Use the capstone folder after completing the core modules.
