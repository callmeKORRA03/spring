# How to Use This Repository

This guide explains how to use the repository depending on your goal.

## If you are learning Spring Boot for yourself

Use this path:

1. Read `START_HERE.md`
2. Read `docs/roadmaps/30-DAY-SPRING-BOOT-ROADMAP.md`
3. Open `docs/INDEX.md`
4. Study the modules in order
5. Run the matching example project after each module
6. Attempt the assignments before checking the answer guide
7. Build your own capstone project at the end

## If you are teaching Spring Boot

Use this path:

1. Read `docs/teacher-edition/TEACHER-GUIDE.md`
2. Read `docs/teacher-edition/3-WEEK-CLASSROOM-PLAN.md`
3. Use each lesson file as your class note
4. Use the assessment section for short tests
5. Use the answer guides for marking support
6. Use the example projects for live teaching demos

## If you want hands on practice

The `examples/` folder contains runnable projects that match the modules.

Suggested order:

- `examples/module-01/hello-rest-api/`
- `examples/module-02/web-core-demo/`
- `examples/module-03/layered-app-demo/`
- `examples/module-04/student-crud-demo/`
- `examples/module-05/api-quality-demo/`
- `examples/module-06/security-starter-demo/`

## If you want interview preparation

Use:

- `docs/module-06-industry-readiness/lesson-36-capstone-and-interview-recap.md`
- `docs/interview-prep/SPRING-BOOT-INTERVIEW-QA.md`

## If you want to build a final project

Use:

- `docs/capstone/CAPSTONE-GUIDE.md`
- `docs/capstone/CAPSTONE-RUBRIC.md`
- `docs/capstone/CAPSTONE-CHECKLIST.md`

## Recommended study rhythm

A simple rhythm that works well is:

- one lesson reading session
- one code walkthrough session
- one small practice session
- one review session

## Best habit for this repository

Do not only read.

For each module:

- read the lesson
- explain it in your own words
- run the example
- change something small in the example
- complete the assignment

## Main idea

This repository is meant to be used like a handbook, a classroom guide, and a practical coding lab at the same time.
