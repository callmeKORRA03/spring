# Module 06 Answer Guide

This file contains suggested answers for the assessments in Module 06.

## Lesson 31 suggested answers

1. Spring Security is a Spring tool used to help protect applications.
2. Backend security matters because applications often handle sensitive data and important actions.
3. A public endpoint can be accessed without login, while a protected endpoint needs authentication or permission.
4. Examples of protected endpoints include admin dashboards, delete actions, and private user records.

## Lesson 32 suggested answers

1. Authentication means checking who a user is.
2. Authorization means checking what a user is allowed to do.
3. The key difference is identity versus permission.
4. A real life example is showing an ID card to enter a building, then using access level to enter different rooms.

## Lesson 33 suggested answers

1. Packaging means preparing the application into a form that can be run or shared more easily.
2. A JAR file is a packaged Java application archive.
3. A common Maven packaging command is `mvn clean package`.
4. A common run command is `java -jar target/app-name.jar`.

## Lesson 34 suggested answers

1. Deployment means making the application available in an environment where others can use it.
2. Deployment matters because an app is not fully useful if it only works on the developer's machine.
3. One difference is that local running is on the developer's machine, while deployed running is in an environment meant for wider access.
4. Examples include cloud servers, virtual machines, container platforms, and company servers.

## Lesson 35 suggested answers

1. Docker is a tool used to package applications and their environment in a portable way.
2. Containers are useful because they help applications run more consistently across different environments.
3. A Dockerfile is a file that gives Docker instructions for building the application image.
4. Docker is useful for backend deployment because it helps improve consistency and portability.

## Lesson 36 suggested answers

1. A capstone project is a final project that brings many learned skills together.
2. A capstone is useful because it helps prove that the learner can combine concepts into one complete backend project.
3. A beginner Spring Boot capstone should include items such as entities, repositories, services, controllers, CRUD endpoints, validation, and exception handling.
4. Common junior interview topics include Spring Boot basics, controllers, dependency injection, entities, repositories, CRUD, DTOs, validation, and `ResponseEntity`.

## Marking guide idea

### Written answers

- full correct answer = 2 marks
- partly correct answer = 1 mark
- wrong or empty answer = 0 marks

### Practical tasks

For industry readiness tasks, check:

- understanding of public versus protected endpoints
- ability to explain authentication and authorization clearly
- clear packaging and deployment awareness
- simple Docker understanding
- realistic capstone planning
- basic interview readiness

## Teacher note

At this stage, confidence and clarity matter a lot.
Students do not need to be experts in security or deployment yet, but they should understand the core ideas and be able to talk about them sensibly.
