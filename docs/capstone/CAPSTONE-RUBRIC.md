# Capstone Rubric

Use this rubric to evaluate a final Spring Boot capstone project.

## Total idea

This rubric is designed for beginner to junior level projects.
It focuses on clarity, structure, correctness, and practical use of Spring Boot concepts.

## 1. Project Structure

### Strong
- clear package structure
- controller, service, repository separated well
- readable naming

### Weak
- all logic crowded into very few files
- poor naming
- no clear structure

## 2. CRUD Functionality

### Strong
- create, read, update, and delete all work
- routes are clear and consistent

### Weak
- missing major CRUD actions
- inconsistent endpoint design

## 3. Data Modeling

### Strong
- entity is sensible
- primary key is clear
- field types make sense

### Weak
- entity design is confusing
- poor field choice
- unclear identifier strategy

## 4. Validation

### Strong
- request data is checked properly
- invalid input is handled clearly

### Weak
- accepts obviously bad data
- validation missing or inconsistent

## 5. Exception Handling

### Strong
- missing records handled clearly
- error responses are useful

### Weak
- bad crashes or unclear error output
- exceptions ignored

## 6. DTO Usage

### Strong
- DTOs used sensibly for request or response control

### Weak
- returns internal data carelessly without thinking

## 7. Service Layer Quality

### Strong
- business logic belongs in services
- controllers stay clean

### Weak
- controllers contain too much logic
- service layer unclear or missing

## 8. Configuration and Project Quality

### Strong
- configuration is organized
- README is helpful
- project is easy to understand and run

### Weak
- poor run instructions
- messy setup
- no clear project explanation

## 9. Documentation

### Strong
- API is documented clearly
- example requests are included

### Weak
- endpoints are hard to understand
- no usage instructions

## 10. Presentation and Explanation

### Strong
- learner can explain what the project does
- learner understands the design choices

### Weak
- learner cannot explain the project clearly
- project appears copied without understanding

## Suggested simple scoring idea

- 0 = missing
- 1 = weak
- 2 = fair
- 3 = good
- 4 = strong
- 5 = excellent

## Main rule

A good capstone is not just one that works.
A good capstone is one the learner can explain, defend, and improve.
