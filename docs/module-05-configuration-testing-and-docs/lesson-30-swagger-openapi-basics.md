# Lesson 30: Swagger and OpenAPI Basics

## Lesson objective

By the end of this lesson, you should be able to:

- understand what API documentation means
- explain OpenAPI in simple English
- explain what Swagger UI is used for
- understand why documentation is useful in backend development

## Why API documentation matters

When an API grows, developers need a clear way to understand:

- available endpoints
- request bodies
- response formats
- status codes

Without documentation, using the API becomes harder.

## What is OpenAPI?

OpenAPI is a standard for describing APIs clearly.

## What is Swagger UI?

Swagger UI is a user friendly interface that can show API documentation in a browser.

It often lets developers:

- see endpoints
- inspect request and response shapes
- test endpoints quickly

## Simple classroom explanation

Say this in class:

OpenAPI helps describe an API clearly, and Swagger UI helps us view and try that documentation easily.

## Why this is useful

API documentation helps:

- backend developers
- frontend developers
- testers
- new team members

## Real life analogy

Think of a user manual for a machine.

The machine may work, but without a manual it is harder to use correctly.
An API manual is similar.

## Common mistakes beginners make

### 1. Thinking documentation is optional forever
Documentation becomes more important as projects grow.

### 2. Thinking only external users need documentation
Even your own team benefits from it.

### 3. Waiting too long before learning documentation tools
It is good to understand the purpose early.

## Beginner setup idea

Many Spring Boot projects use a dependency that provides Swagger UI for OpenAPI based documentation.

Then the documentation can usually be opened in the browser.
