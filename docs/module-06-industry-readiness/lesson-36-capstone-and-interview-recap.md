# Lesson 36: Capstone Project and Interview Style Recap

## Lesson objective

By the end of this lesson, you should be able to:

- understand the purpose of a capstone project
- connect the earlier modules into one project idea
- identify topics commonly discussed in junior Spring interviews
- prepare for the next stage of growth after the handbook

## What is a capstone project?

A capstone project is a final project that brings many learned skills together.

It helps show that you can do more than isolated lessons.

## Why a capstone matters

A capstone project helps you practice:

- project structure
- controllers
- services
- repositories
- CRUD
- validation
- exception handling
- configuration
- documentation
- testing mindset

## Simple classroom explanation

Say this in class:

A capstone project is where we combine what we have learned into one complete backend application.

## Good capstone ideas

Examples include:

- student management system
- library management API
- course registration system
- simple inventory API
- task manager API

## What your capstone should include

A strong beginner Spring Boot capstone can include:

- at least one entity
- repository layer
- service layer
- controller layer
- CRUD endpoints
- validation
- exception handling
- DTOs
- basic documentation
- clean package structure

## Interview style recap topics

A junior Spring Boot interview may ask about:

- what Spring Boot is
- difference between `@Controller` and `@RestController`
- what dependency injection means
- what `@Service` is used for
- what an entity is
- what a repository does
- what CRUD means
- what DTO means
- what `ResponseEntity` does
- why validation matters

## Real life analogy

Think of earlier lessons as separate training exercises.
The capstone is the final match where the skills must work together.

## Common mistakes beginners make

### 1. Building a capstone that is too large too early
A smaller clean project is better than an oversized broken one.

### 2. Ignoring structure in the final project
The capstone should show good habits, not just working endpoints.

### 3. Thinking the first capstone must be perfect
It should be useful, clear, and improving. Perfection is not the first goal.
