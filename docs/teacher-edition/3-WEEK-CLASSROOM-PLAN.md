# 3 Week Classroom Plan

This file helps an instructor use the handbook as a teaching plan over 3 weeks.

## Format used

- 3 classes per week
- 2 hours per class
- beginner friendly teaching pace

## Week 1: Foundations and Web Basics

### Class 1
Focus:
- what backend means
- what Spring Boot is
- HTTP basics

Goal:
Students understand the purpose of backend development and the role of Spring Boot.

### Class 2
Focus:
- Maven and `pom.xml`
- project structure
- first Spring Boot app

Goal:
Students can create and run a simple Spring Boot app.

### Class 3
Focus:
- controllers
- first endpoints
- request parameters
- path variables

Goal:
Students can build and test simple endpoints.

## Week 2: Structure and Data

### Class 4
Focus:
- request body
- POST requests
- `ResponseEntity`
- status codes

Goal:
Students understand input and output handling more clearly.

### Class 5
Focus:
- IoC
- Dependency Injection
- services
- constructor injection
- layered architecture

Goal:
Students begin structuring apps more professionally.

### Class 6
Focus:
- JPA
- entities
- repositories
- CRUD idea

Goal:
Students understand the path from entity to repository to API.

## Week 3: Real Backend Thinking

### Class 7
Focus:
- validation
- exception handling
- DTO basics

Goal:
Students understand cleaner data handling.

### Class 8
Focus:
- configuration
- profiles
- logging
- testing basics
- Swagger idea

Goal:
Students understand that a real backend needs more than endpoints.

### Class 9
Focus:
- security basics
- packaging
- deployment idea
- Docker introduction
- capstone recap

Goal:
Students understand the next stage beyond beginner CRUD.

## Suggested teaching pattern for each class

- 20 minutes explanation
- 30 minutes guided walkthrough
- 30 minutes live coding
- 20 minutes student practice
- 10 minutes correction and recap
- 10 minutes assignment briefing

## Strong teacher advice

Do not rush students into advanced security or deployment detail too early.
Make sure they understand request flow, service flow, and CRUD clearly first.

## Main classroom success sign

Students should be able to explain what each layer does, not only type code they memorized.
