# Student Management API Capstone Example

This folder is the suggested capstone destination for learners who want to build a final project from the handbook.

## Purpose

This is not meant to be a huge advanced enterprise system.
It is meant to be a clean beginner to junior level capstone.

## Suggested features

- Student entity
- CRUD endpoints
- validation
- DTOs
- exception handling
- configuration file
- Swagger or manual docs
- optional basic security

## Suggested endpoints

- `POST /students`
- `GET /students`
- `GET /students/{id}`
- `PUT /students/{id}`
- `DELETE /students/{id}`

## Suggested fields

- id
- name
- email
- course

## Suggested package structure

- `controller`
- `service`
- `repository`
- `model`
- `dto`
- `exception`
- `config`

## Suggested stretch goals

- profile based configuration
- logs
- OpenAPI docs
- Dockerfile
- simple authentication

## How to use this folder

Use the earlier example projects as references, then build your own final version here.

## Final goal

A finished capstone in this folder should prove that the learner can build a structured Spring Boot backend independently.
