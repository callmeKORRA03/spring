# How to Use This Repository

This guide helps you use the repository in the best way depending on your goal.

## If you are learning Spring Boot for yourself

Follow this order:

1. Read `START_HERE.md`
2. Read `docs/INDEX.md`
3. Follow `docs/roadmaps/30-DAY-SPRING-BOOT-ROADMAP.md`
4. Study one module at a time
5. Run the example project that matches the module
6. Answer the assessment questions before reading the answer guide
7. Do the assignments with your own hands

## If you are using this as a teacher

Use this order:

1. Read `docs/teacher-edition/TEACHER-GUIDE.md`
2. Open the lesson file for the class topic
3. Use the simple explanations as your teaching note
4. Use the assessment section as in class questions
5. Use the assignment section as student homework or lab practice
6. Use the answer guide when marking or reviewing work

## If you want to move fast

A fast but still useful order is:

- Module 01
- Module 02
- Module 03
- Module 04 example project
- Module 05 key lessons
- Module 06 interview recap

## If you want the deepest value from this repo

Do not only read the notes.

Also:

- run the example projects
- type code manually instead of copying everything
- test the endpoints yourself
- edit the code and see what changes
- create small variations of each project

## Suggested rhythm for self study

A simple pace can be:

- one lesson per day for reading
- one practice session per day for coding
- one review session every weekend

## Suggested rhythm for teaching

A simple class rhythm can be:

- concept explanation
- small code walkthrough
- live run and test
- student attempt
- correction and recap
- assignment

## Important warning

Do not skip the foundations and jump straight to CRUD or security.

The repo is designed in a learning order for a reason.
